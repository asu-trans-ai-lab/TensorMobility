TensorMobility — named workflows

Every workflow: stages → GMNS contract → certificate. Files on disk are the only interface; certificates.json rides beside every output folder. Companion to docs/ECOSYSTEM_DESIGN.md.

W1 generate — self-contained networks

grid / SF / Chicago case spec → unified loader (StaticNetwork contract) → GMNS folder node / link / demand

in: nothing (grid) or data/ registry  ·  out: node.csv link.csv demand.csv

certificate: contract validation (positive times/capacities, zone mapping, demand columns)

W2 assign — certified DTA core

GMNS folder→ column generation (FW / GP / atoms) → full-space Dijkstra pricing audit → path.csv + link_performance.csv

engines escalate on cycling: Picard → MSA → Anderson → stiff-block Newton → NCP/VI

certificate: relative gap (never pool-restricted), conservation, engine + escalation log

W3 import — external kernel audit

TAPLite4MPO / DTALite run → parse link_performance.csv → audit: gap at kernel times, conservation → re-price columns → path.csv

kernels compute; TensorMobility certifies. Cross-validation vs W2 (gap-vs-gap, flow correlation) for free.

certificate: kernel gap, flow correlation vs internal solver, materialized-column coverage

W4 calibrate — dynamic ODME profile

counts / speeds (+ registry corridor) → observability gate (G0) → departure profile φ(t) + bounded low-rank ΔOD → typed-chain projected gradient

the dynamic-odme-lab structure (A = M·Δ·R) as a TensorMobility profile; stages 0–6 are the benchmark ladder

certificate: observability rank, FD-checked gradient, synthetic-recovery gate before any real-data claim

W5 visualize — gui4gmns dashboards

any run folder→ export_gmns() MOE / OD / space-time layers → gui4gmns HTML / Qt / GL → dashboard + certificate panel
certificate: rendered, not just stored — the dashboard shows its own audit table

W6 teach — executable ladder

notebooks T1–T7 → run → certify → explain → CI (nbmake) keeps them honest

T4 pool audit, T5 compression boundary, T7 cycling engine lesson are deliberate failure-mode classes

W7 profile — layered equilibria

extension axes (company / vtype / stage / match…) → blocks + shared cells + price sync → Rx ≤ Sy · MAGE · multilayer freight
certificate: complementarity + patience/fleet feasibility + multi-start spread; duality gaps reported, never assumed zero
