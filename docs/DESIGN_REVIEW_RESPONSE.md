## One respectful disagreement, stated openly

§6.1 recommends replacing the four pages with one TS/Svelte app.
Agreed as the end state, but the single-file pages are currently a
feature for the offline/teaching distribution model (double-click to
open, no build step — the same property the v1/v3 design docs
required). Proposed sequencing: first the shared **scenario JSON**
(kills data drift, the worst duplication), then extract the shared
algorithms into one included JS file (kills algorithm drift), and
only then consider the app framework once certified-replay mode
exists. This keeps the anti-drift goal while preserving the
zero-install property until there is a build pipeline worth its cost.
